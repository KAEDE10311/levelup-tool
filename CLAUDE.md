# levelup - 学習管理ツール

個人学習を記録・管理するためのツール。

## 技術スタック
- React + TypeScript
- Vite
- Zustand（状態管理）
- React Hook Form + Zod（フォーム・バリデーション）
- Tailwind CSS v4
- react-icons
- recharts（グラフ描画）

## コーディング方針
- 関数コンポーネント + Hooksで統一
- 型定義は明示的に書く（anyは使わない）
- Tailwindのユーティリティクラスでスタイリング（個別CSSファイルは基本作らない）
- Zustandのstoreはfeature単位で分割予定
- 変数・関数・コンポーネントには役割がひと目でわかるようコメントを付ける（「何をしているか」ではなく「何のためにあるか」を1行で）
  - 型定義：各フィールド／各interfaceが表すものにコメント
  - 関数：入力から出力への変換内容や、使われる場面にコメント
  - コンポーネント：ページ内でのどの部分を担当するかにコメント
  - 変数：他の値との関係（何から作られ、どこで使われるか）にコメント
- 上記のコメントを付けた型・関数・コンポーネント・変数は、`docs/glossary.md`にもファイル別の表（識別子／役割）として追記する
  - コード側のコメントが正、`docs/glossary.md`はその写しという位置付け
  - 新規追加・役割の変更・削除をしたら、コードのコメントを直した上で`docs/glossary.md`の該当行も同じタイミングで更新する

## ディレクトリ構成
```
src/
  components/
    dashboard/   # メインページ（今日の学習状況）用のコンポーネント
  pages/         # ページ単位のコンポーネント（DashboardPageなど）
  lib/           # 集計・色割り当てなどの純粋関数
  data/          # ダミーデータ（記録機能実装後、実データに置き換え予定）
  types/         # 型定義
```
- `pages/` がルーティング単位、`components/xxx/` はそのページ専用の部品を置く
- `lib/` にはUIを持たないロジックのみ（テストしやすさ優先）
- 現状データは`data/mockStudyRecords.ts`のダミーデータ。記録入力機能の実装後はZustand storeに置き換える

## ブランチ運用
- main：常に動く状態を保つ
- feature/xxx：機能単位で作業ブランチを切る
